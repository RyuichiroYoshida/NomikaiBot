## 改善案

README の MVP は、たとえば次のように少し具体化すると開発に入りやすいです。

### MVP

- 主催者が `/nomikai create` で飲み会募集を作成できる
  - 入力項目: タイトル、日時、場所、説明、定員、締切
- Bot が募集メッセージを指定チャンネルに投稿する
  - メッセージには参加・取消ボタンを表示する
- 参加者が募集メッセージから参加・取消できる
- 主催者が `/nomikai participants` で参加者一覧を確認できる
- 参加者が募集メッセージから日時、場所、主催者、参加者数を確認できる

## 次に取り組むべきタスク

1. MVP のコマンド一覧を `Docs/CommandDoc.md` に書く  
2. 募集作成時の入力項目を決める  
3. 参加・取消・締切後の挙動を決める  
4. 主催者と参加者の権限ルールを決める  
5. MySQL に保存するテーブル候補を整理する
